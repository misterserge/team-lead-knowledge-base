# TTI (Time to Interactive)

**TTI (Time to Interactive)** — это метрика производительности фронтенда, показывающая, через сколько времени после загрузки страницы она становится полностью интерактивной.

То есть пользователь может кликать, вводить данные и интерфейс стабильно реагирует.

---

## Как измеряется

Измеряется в секундах с момента начала загрузки страницы.

### TTI включает

- Загрузку HTML, CSS, JS, изображений
- Выполнение JavaScript (в том числе блокирующего)
- Отрисовку основного контента
- Готовность обработчиков событий

---

## Пример

Страница загрузилась визуально за 1.5 с, но скрипты ещё выполняются, и клики не работают.

Интерфейс реально "оживает" только на 3-й секунде.

→ **TTI = 3 с**

---

## Почему важно

- Это одна из ключевых метрик UX (входит в Lighthouse / Core Web Vitals)
- Высокий TTI означает, что сайт "выглядит загруженным", но по факту "тормозит"
- Оптимизация TTI напрямую повышает конверсию и удовлетворённость пользователей

---

## Как оптимизировать

| Метод | Описание |
|-------|----------|
| **Lazy-load и code splitting** | Загружать только нужный код |
| **Defer / async для скриптов** | Не блокировать парсинг HTML |
| **Web Workers** | Тяжёлые вычисления выносить в отдельный поток |
| **Critical CSS** | Инлайнить только критические стили |
| **Минимизация работы main thread** | Анализировать через Chrome DevTools → Performance |

---

## Связь с другими метриками

TTI входит в **Core Web Vitals** вместе с:

- **LCP** (Largest Contentful Paint) — скорость загрузки основного контента
- **FID** (First Input Delay) — задержка первого взаимодействия
- **CLS** (Cumulative Layout Shift) — стабильность верстки

> Все эти метрики измеряются в Lighthouse и влияют на ранжирование в Google.
