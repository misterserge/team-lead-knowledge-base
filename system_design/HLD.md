# High-Level Design (HLD)

**HLD** — это документ или этап проектирования, где описывается архитектура системы на высоком уровне без углубления в детали реализации.

## Основные характеристики

- **Фокус:** на архитектуре и взаимодействии компонентов, а не на коде
- **Аудитория:** тимлиды, архитекторы, заказчики, инженеры
- **Форма:** диаграммы, схемы, описание модулей и их связей

## Что включает HLD

| Раздел | Описание |
|--------|----------|
| Архитектурные принципы | Монолит, микросервисы, event-driven и т.д. |
| Основные модули | Компоненты системы и их назначение |
| Потоки данных | Откуда и куда идут данные |
| Базы данных | Используемые БД и их назначение (без подробных схем) |
| Интеграции | Связи с внешними системами |
| Диаграммы | C4-модель (Context, Container) |
| NFR | Масштабируемость, отказоустойчивость, безопасность |

## HLD vs LLD

**HLD** обычно предшествует **LLD (Low-Level Design)**, где уже описываются детали реализации:

- Структуры таблиц БД
- Диаграммы классов
- API-эндпоинты
- Конкретные алгоритмы
