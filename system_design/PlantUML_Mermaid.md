# PlantUML vs Mermaid

**PlantUML** и **Mermaid** — два DSL (языка описания диаграмм), которые решают похожие задачи, но отличаются синтаксисом, экосистемой, возможностями и способом рендеринга.

## Главное отличие

- **PlantUML** → мощнее, старше, поддерживает больше видов диаграмм, ориентирован на UML и архитектуру
- **Mermaid** → проще, нативно работает в Markdown/HTML/MDX/GitHub, ориентирован на веб и документацию

---

## Подробное сравнение

### 1. Синтаксис и сложность

| PlantUML | Mermaid |
|----------|---------|
| Очень выразительный DSL | Минималистичный, простой синтаксис |
| Ближе к UML-парадигме | Сфокусирован на популярных типах диаграмм |
| Иногда выглядит "тяжелее" | Проще читать и писать |
| Позволяет делать огромные и сложные диаграммы | Идеален для документации |

### 2. Поддерживаемые диаграммы

**PlantUML** — поддерживает почти ВСЕ UML диаграммы:

- Class
- Sequence
- Component
- Deployment
- Use Case
- Activity
- State
- Object
- **C4-модель** (встроенная!)
- ArchiMate
- ERD
- Gantt
- WBS (Work Breakdown Structure)
- Mindmap
- Graphviz integration

> PlantUML = огромный арсенал для solution architecture

**Mermaid** — поддерживает:

- Sequence
- Flowchart
- Class Diagram
- ER Diagram
- State Diagram
- Gantt
- Pie Chart
- Git graph
- Journey chart
- Timeline

> Нет Deployment diagram, нет полноценной C4 поддержки (только костылями), нет ArchiMate

### 3. Рендеринг

**PlantUML:**
- Требует Java + Graphviz (но есть серверы/облако)
- Рендеринг точнее и глубже (особенно для UML)
- Можно запускать локально, в CI/CD, в Docker

**Mermaid:**
- Рендерится прямо в браузере (JS)
- GitHub, GitLab, Obsidian, Notion — все поддерживают Mermaid native
- Идеален для Markdown

### 4. Экосистема и интеграции

**PlantUML:**
- Используется архитекторами, backend-системами и enterprise
- IDE плагины: IntelliJ, VSCode, Eclipse
- Хорош для больших проектов и системного проектирования

**Mermaid:**
- Абсолютный стандарт для документации в GitHub/Notion/Confluence/MDX
- Идеален для доступности и быстрой визуализации

### 5. Гибкость и кастомизация

- **PlantUML:** множество тем, стилизация, шейпы, кастомные спрайты, автоматизация диаграмм на основе данных
- **Mermaid:** ограниченная стилизация, минимализм

### 6. Крупные диаграммы и большие схемы

- **PlantUML** → отлично масштабируется на огромные sequence diagrams и C4 модели
- **Mermaid** → sequence диаграммы крупных форматов становятся нечитаемыми

---

## Что использовать?

### Используй PlantUML, если:

- Делаешь UML/architecture документацию
- C4 модель (Context/Container/Component/Code)
- Сложные Sequence диаграммы
- Надо интегрировать в архитектурные ADR
- Хочешь глубокую детализацию и контроль

### Используй Mermaid, если:

- Пишешь документацию в GitHub/Notion/Confluence
- Нужен быстрый рендер в браузере
- Нужны flowcharts, timelines, простые sequence diagrams
- Хочешь минимальный overhead

---

## Резюме

| Аспект | PlantUML | Mermaid |
|--------|----------|---------|
| **Назначение** | Профессиональные архитектурные и UML-диаграммы | Легкие и быстрые диаграммы прямо в Markdown |
| **Сложность** | Выше | Проще |
| **Рендеринг** | Java + Graphviz | Браузер (JS) |
| **GitHub native** | Нет | Да |
| **C4 модель** | Встроена | Костыли |

# ArchiMate диаграммы

**ArchiMate** — это язык архитектурного моделирования, созданный специально для **Enterprise Architecture (EA)**, чтобы описывать бизнес, приложения и технологическую инфраструктуру единым стандартным способом.

> ArchiMate связывает бизнес-уровень, данные, приложения, сервисы и инфраструктуру в одну логическую модель.

---

## Основные особенности

### 1. Связующее звено между бизнесом и IT

Главная идея: показать, как **бизнес-процессы → опираются на приложения → которые работают на инфраструктуре**.

То, что UML/C4/Flowcharts не покрывают.

### 2. Иерархия уровней (tri-layer)

ArchiMate строго делит архитектуру на 3 слоя:

#### Business Layer
- Бизнес-актеры
- Роли
- Бизнес-процессы
- Бизнес-функции
- Бизнес-сервисы

#### Application Layer
- Приложения
- Модули
- Компоненты
- Интеграции
- App-services

#### Technology Layer
- Серверы
- Базы данных
- Сети
- Устройства
- Технологические сервисы

Каждый уровень описывается одним языком, что позволяет делать end-to-end модели.

### 3. Четкая система связей между элементами

ArchiMate определяет семантически строгие связи:

- **flows** — потоки данных/взаимодействия
- **assignment** — назначение
- **realization** — реализация
- **serving** — обслуживание
- **composition** — композиция
- **aggregation** — агрегация
- **triggers** — триггеры
- **access** — доступ

Благодаря этому можно формально показать:
- Кто вызывает кого
- Что зависит от чего
- Какой процесс требует какой сервис
- Какое приложение реализует какой бизнес-процесс
- Какая технология поддерживает какое приложение

### 4. Язык на уровне предприятия (enterprise)

В отличие от UML или C4, которые описывают систему, ArchiMate описывает **целую организацию**:

- Бизнес-способности (capabilities)
- Value streams
- Организационную структуру
- Доменные модели
- Процессы и value chain
- Аппликативный ландшафт
- Технологический ландшафт
- Data flow между доменами

> Это именно EA язык, а не язык программной архитектуры.

### 5. Стандартизированные типы элементов

Каждый элемент имеет фиксированную форму и роль:

- Business Process
- Business Actor
- Application Component
- Application Interface
- Data Object
- Node
- Device
- System Software

Это способствует единообразию в документации.

### 6. Viewpoints (виды)

ArchiMate вводит понятие "viewpoints" — готовые срезы модели для разных ролей:

- Application Cooperation View
- Business Process View
- Technology Infrastructure View
- Data Flow View
- Layered View
- Motivation View
- Strategy View

Каждый viewpoint — это как преднастроенный "фильтр" на общую модель.

### 7. Часть TOGAF

ArchiMate стандартизован **The Open Group**, как и TOGAF.  
Поэтому он идеально стыкуется с ADM (Architecture Development Method).

---

## ArchiMate vs UML / C4 / BPMN

| Язык | Для чего | Уровень |
|------|----------|---------|
| **ArchiMate** | Enterprise Architecture, связи бизнес → приложения → технологии | Стратегический / корпоративный |
| **UML** | Детальные дизайн-модели для разработчиков | Системный, технический |
| **C4** | Коммуникационные архитектурные диаграммы | High-level архитектура |
| **BPMN** | Бизнес-процессы | Бизнес-уровень |

> ArchiMate перекрывает ВСЕ уровни, но без избыточных технических деталей.

---

## Резюме

**ArchiMate** — стандартный язык для моделирования всей архитектуры компании: бизнес → приложения → инфраструктура — единым, строгим семантическим способом.
